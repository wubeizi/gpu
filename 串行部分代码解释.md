## 项目结构
```
bfs_project/
├── CMakeLists.txt                # 构建配置，含CUDA扩展注释
├── benchmark.sh                  # 批量测试脚本（跑4个规模）
├── run.sh                        # 单次构建运行脚本
├── build/                        # 编译输出目录
├── graph_random_16_d16.bin       # 2^16规模的图数据（群里压缩包提供）
├── graph_random_18_d16.bin       # 2^18规模的图数据
├── graph_random_20_d16.bin       # 2^20规模的图数据
├── graph_random_22_d16.bin       # 2^22规模的图数据
├── include/
│   ├── graph.h                   # CSR图结构定义 + 图生成/保存/加载接口
│   ├── bfs.h                     # BFS接口（串行已实现，GPU版本已预留注释）
│   └── utils.h                   # Timer计时器 + check_result正确性验证 + print_performance输出
└── src/
    ├── main.cpp                  # 主程序，GPU版本测试代码已写好注释模板
    ├── graph.cpp                 # 随机图生成 + CSR二进制文件保存/加载
    ├── bfs_cpu.cpp               # CPU串行BFS实现
    └── utils.cpp                 # 计时、验证、输出工具实现
```

## 编译运行
```bash
# 仅CPU版本
g++ -std=c++17 -O2 -I include src/main.cpp src/graph.cpp src/bfs_cpu.cpp src/utils.cpp -o build/main
./build/main 20          # scale=20, avg_degree=16, source=0

# 或用脚本
./benchmark.sh           # 跑 2^16, 2^18, 2^20, 2^22 四个规模
```

生成的四种图在群里的压缩包中名字是 `graph_random_{scale}_d16.bin`，按照第一部分项目结构所示解压后放在项目下，运行代码 `./benchmark.sh` 直接用已经生成的图结构并基于此继续实验。

## 如何添加GPU版本

1. 新建 `.cu` 文件，如 `src/bfs_gpu_naive.cu`，实现函数：
```cpp
#include "bfs.h"
#include "utils.h"

double bfs_gpu_naive(const CSRGraph& graph, int source, std::vector<int>& level) {
    // 把 graph.row_offsets 和 graph.col_indices 用 cudaMemcpy 传到设备端
    // 实现你的 kernel
    // 把结果拷回 level
    // 用 CUDA Event 计时，返回微秒数
}
```

2. 在 `bfs.h` 中取消对应函数声明的注释。
3. 在 `main.cpp` 中取消对应测试代码块的注释。
4. 修改 `CMakeLists.txt`：取消底部CUDA部分的注释，把你的 `.cu` 文件加进去，`CMAKE_CUDA_ARCHITECTURES` 改成你的显卡对应值（RTX 4090=89, RTX 3090=86, RTX 2080=75）。
5. 用CMake编译：
```bash
mkdir -p build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)
./main 20
```

## 输出格式

程序会自动输出执行时间、加速比、正确性验证：
```
GPU_naive Execution Time elapsed 1234 us
GPU_naive speed up 65.432X
[GPU_naive] Check result success!
```

加速比 = CPU串行时间 / 你的GPU时间。正确性验证是和CPU结果逐节点对比level值。
